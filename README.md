# AI Circuit Design Assistant — Dataset Generator (RC LPF)

> This repository contains the dataset-generation tools for the AI Circuit Design Assistant.
> The current scripts generate SPICE-based datasets for analog topologies (starting with RC low-pass filters).
> Teammates can add generators for other topologies (RLC, amplifiers, filters, etc.) and contribute training data.

---

## Project structure (recommended)
AI_Circuit_Assistant/
├─ src/
│ ├─ generate_rc_dataset_ngspice.py # main dataset generator for RC LPF
│ └─ parse_and_check.py # helper/debug scripts
├─ sim/ # autogenerated: .cir and .dat files
├─ data/ # autogenerated: circuits.csv
├─ models/ # trained models (saved .pkl or .pt)
├─ ui/ # frontend code (optional)
├─ notebooks/ # EDA and plots
├─ requirements.txt
└─ README.md


---

## Quick overview
- The pipeline automates SPICE simulations (Ngspice) to produce `(component values) -> (performance)` pairs.
- Current implementation: **RC low-pass filter** dataset
  - For each (R, C) pair it runs `.ac dec` sweep and extracts −3 dB cutoff.
- Output: `data/circuits.csv` with columns: `id,topology,R_ohm,C_f,cutoff_hz`

## Prerequisites (what you/teammates need)
- Python 3.8+ (recommended 3.10+)
- Ngspice (we use the console binary `ngspice_con.exe` on Windows)
use this link ->  https://sourceforge.net/projects/ngspice/#
- Python packages: see `requirements.txt` (install with `pip install -r requirements.txt`)

**Important**: Do **not** commit Ngspice binary (large proprietary or platform-specific). Instead, have teammates download and place `ngspice_con.exe` in a local folder and set the path in `generate_rc_dataset_ngspice.py` (the `NGSPICE_EXE` variable).
add the path of ngspice_con.exe in environment varibles->user variables-> path->edit->add new-> ok
