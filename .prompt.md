# Reusable Prompt for Circuit Diagram Improvements

Use this prompt for any topology by replacing placeholders.

## Full Prompt

Update the schemdraw visualization for topology: {topology_name}.

Goal:
Produce a clean, professional, production-ready circuit image with accurate topology, readable labels, and zero overlap.

Requirements:
1. Topology correctness
- Draw the real textbook topology for {topology_name}, not an abstract block sketch.
- Ensure all branches and return paths are electrically connected and visually clear.

2. Layout and scale
- Make the circuit larger and visually attractive (spacious, balanced composition).
- Keep the circuit on the left and the info panel on the right.
- Avoid cramped geometry and avoid excessive empty space.

3. Label placement
- Place each value close to its corresponding component so correspondence is obvious.
- Never let labels overlap components, wires, nodes, or other labels.
- If overlap occurs, automatically reposition labels with small offsets until clean.
- Do not solve overlap by pushing labels too far away.

4. Units and formatting
- Show units beside values (for example: V, Hz, F, ohm).
- Use compact engineering notation when useful (for example: kHz, mF, kohm) to reduce long text overlap.
- Keep labels human-readable and consistent.

5. Right-side panel formatting
- Show Inputs and Predicted Outputs in a clean right-side panel.
- Left-align all panel text.
- Use consistent key:value alignment and spacing.
- Ensure Inputs and Predicted Outputs blocks never overlap.

6. Background/export
- Export with solid white background (not transparent).
- Preserve high quality output.

7. Output and validation
- Save to: {output_path}
- Run a final visual check:
  - topology correct
  - larger and balanced
  - no overlap anywhere
  - labels close to the right components
  - units present
  - right panel clean and left-aligned
  - white background confirmed

Inputs to use:
{inputs_dict}

Predicted outputs to display:
{outputs_dict}

Optional topology-specific rule:
- If topology has mode/type flags (for example clamper_type), orient/arrange polarity-sensitive components accordingly.

## Quick Prompt (Short Version)

Improve schemdraw for {topology_name} with textbook-accurate topology and zero overlap.
Make layout spacious: circuit left, info panel right.
Place labels near corresponding components, auto-nudge to avoid overlap, keep labels readable.
Show units and compact engineering notation.
Right panel must include Inputs and Predicted Outputs, left-aligned with consistent key:value spacing and no block overlap.
Export solid white background to {output_path}.
Use:
Inputs: {inputs_dict}
Outputs: {outputs_dict}
Apply polarity/mode rules when topology includes type flags.
